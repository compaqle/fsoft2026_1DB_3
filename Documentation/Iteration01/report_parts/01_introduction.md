# Introdução

O presente relatório técnico documenta o processo de análise e especificação de um sistema de software para a gestão de um supermercado. Este sistema visa automatizar tarefas críticas como o controlo de inventário, a gestão de funcionários (caixas) e o processamento de vendas em tempo real.

A motivação para este projeto reside na necessidade de criar uma solução robusta, modular e extensível que suporte operações diárias de retalho, garantindo a integridade dos dados e uma interface de utilizador intuitiva via linha de comandos (CLI).

Este documento está estruturado da seguinte forma: a Secção 2 detalha a análise de requisitos (User Stories e Requisitos Não Funcionais); a Secção 3 apresenta o Modelo de Domínio e o Glossário; a Secção 4 descreve o Modelo de Casos de Uso e Diagramas de Sequência de Sistema (SSD); e, por fim, a Secção 5 apresenta as conclusões desta primeira iteração.
